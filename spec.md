2. The Logic Diagnostic Engine (Helping You Map Metrics)
To prove when a child is writing code quickly but lacks logical comprehension, the backend will process the JSON logs against three core behavioral indicators:
Metric A: The Paste Shockwave (Exploitative Behavior)
• How to detect it: An insert or paste action where the delta length is greater than 40 characters, but the timeSpentMs is less than 10ms.
• The Cognitive Diagnostic: The student is likely relying heavily on copy-pasting from an AI chatbot window or a friend, bypassing the syntax execution loop entirely.
Metric B: The Backspace Cascade (Blind Guessing)
• How to detect it: Immediately following an execution with a failed status, look at the next events. If a delete_cascade or high volume of backspaces wipes out the block within a 3-second window, trigger this indicator.
• The Cognitive Diagnostic: The student doesn't understand why the error occurred. Instead of debugging or reading the stack trace, they panic, erase their progress, and try an entirely different random structure.
Metric C: The Infinite Execution Loop (Trial and Error)
• How to detect it: A window of 3 consecutive execution events within 60 seconds where the code snapshots have a Levenshtein distance (or string diff change) of less than 3 characters.
• The Cognitive Diagnostic: The child is changing random symbols (changing a + to a -, or changing a variable name) and spamming the run button, hoping the system gives them a green checkmark without understanding the data flow.
3. Core Outcomes Layout
View 1: The Teacher's Panel (Actionable Interventions)
Instead of forcing teachers to watch 30 video streams at once, this dashboard flags micro-anomalies:
• Alert Queue: * ⚠️ “Kevin’s session just detected a Backspace Cascade. He has hit execution errors 4 times and deleted all code each time. He is stuck and needs conceptual guidance on loop limits.” • 🛑 “Alice just triggered a Paste Shockwave (85 lines in 2ms). Intercept her workspace to verify comprehension.”
• Class Matrix: A live grid showing Typing Velocity vs. Logical Stability Score. Students in the bottom-right quadrant (High Typing Velocity, Zero Logical Stability) are immediately highlighted for teacher support.
View 2: The Parent's Progress Portal (Demonstrating Value)
Parents don't want to look at a raw terminal or abstract syntax errors. They want to see tangible progress:
• The Timeline Replay: A mini interactive playback canvas where parents can hit a "Play" button to watch their child's code assemble over time.
• Cognitive Mastery Badges (The Positives): * 🧠 "Resilient Debugger": Earned because the student spent 45 seconds reading a NameError block and corrected the variable instead of wiping out the file. • 🏗️ "Architectural Thinker": Earned because their typing speed was steady, showing intentional composition over erratic typing.
🛠️ Architecture Setup for the Prototype
1. Frontend Sandboxed Workspace: A single-page application using Vite + TypeScript. Embed a standard text area or a light setup like the Monaco Editor (the core engine behind VS Code). Add an event hook onto changes that batches these entries, formatting them into the Telemetry JSON payload.
2. Backend Engine: A lightweight Node.js/Express server or Python/FastAPI system. When the telemetry batch arrives, it writes directly to a filesystem layout: /data/classes/{classId}/{studentId}_session.json.
3. Analytical Rules Script: A simple parsing script that reads the updated JSON file on change, applies the simple rules for the metrics outlined above, and computes an updated score.
